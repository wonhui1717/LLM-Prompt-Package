# 01. LLM 모델 비교·선정 보고서

# 1. 비교 목적

본 프로젝트의 목적은 동일한 업무 과업을 3종 이상의 대규모 언어모델(LLM)에 적용하여 성능을 비교하고, 연구보고서 개요 자동 생성 업무에 가장 적합한 모델을 선정하는 것이다.

비교는 동일한 입력 프롬프트를 사용하여 수행하였으며, 결과의 정확성, 논리성, 형식 준수, 환각(Hallucination) 억제 능력 등을 중심으로 평가하였다.

---

# 2. 업무 과업 정의

## 업무 과업

**연구보고서 개요 자동 생성**

## 업무 설명

사용자가 연구 주제를 입력하면 AI가 연구 목적, 연구 필요성, 핵심 키워드, 보고서 목차, 각 장의 핵심 내용 및 기대효과를 포함한 학술적인 연구보고서 개요를 자동 생성한다.

---

# 3. 타겟 사용자

- 신소재공학과 학부생
- 대학원생
- 정부출연연구원 연구원
- 기업 연구개발(R&D) 연구원

---

# 4. 입력 데이터 형태

사용자는 다음 정보를 입력한다.

- 연구 주제
- 출력 형식(일반형/표형)
- 추가 요구사항
- 연구 목적(선택)

예시

```
연구 주제
DED 공정에서 TiB₂ 첨가가 Al6061의 미세조직과 기계적 특성에 미치는 영향

출력 형식
표 형식

추가 요구사항
실험 결과는 추측하지 말 것
```

---

# 5. 출력 규격

AI는 반드시 다음 순서대로 출력한다.

1. 보고서 제목
2. 연구 목적
3. 연구의 필요성
4. 핵심 키워드
5. 보고서 목차
6. 각 장의 핵심 내용
7. 기대 효과

정보가 부족한 경우에는 반드시 **"확인 필요"** 를 표시한다.

---

# 6. 동일 입력 프롬프트

본 비교에서는 아래 프롬프트를 모든 모델에 동일하게 입력하였다.

```text
당신은 30년 경력의 정부출연연구원 연구부장이다.

다음 연구 주제를 바탕으로 학술 보고서의 개요를 작성하시오.

[연구 주제]
DED 공정에서 TiB₂ 첨가가 Al6061의 미세조직과 기계적 특성에 미치는 영향

다음 요구사항을 반드시 준수하시오.

1. 보고서 제목
2. 연구 목적
3. 연구의 필요성
4. 핵심 키워드
5. 보고서 목차
6. 각 장의 핵심 내용
7. 기대 효과
8. 실제 실험 데이터나 논문에 없는 내용은 추측하지 말고 일반적인 연구 방향으로 기술한다.
```

# 8. 평가 기준

본 프로젝트는 다음 6가지 평가 기준으로 비교하였다.

1. 정확성
2. 논리성
3. 형식 준수
4. 한국어 자연스러움
5. 환각(Hallucination) 억제
6. 업무 자동화 적합성

평가는 1점(매우 낮음)부터 5점(매우 우수)까지 부여하였다.

# 9. 모델별 결과 요약

## ChatGPT

### 장점

- 출력 형식을 가장 안정적으로 유지하였다.
- 연구보고서 형식을 일관되게 작성하였다.
- "확인 필요" 기능을 통해 정보 부족 상황을 명확하게 표시하였다.
- 반복 사용 시 결과의 일관성이 높았다.

### 단점

- Claude보다 전공 분야 설명이 다소 일반적이었다.

---

## Claude

### 장점

- 재료공학 전문성이 가장 우수하였다.
- 대학원 논문 수준의 문체를 보여주었다.
- 이론적 배경 설명이 풍부하였다.

### 단점

- 일부 내용을 일반적인 사실처럼 단정적으로 작성하였다.
- 정보 부족 시 확인 질문 기능이 부족하였다.

---

## Gemini

### 장점

- 다양한 전문 용어를 활용하였다.
- 연구 방향을 적극적으로 제시하였다.

### 단점

- 요구하지 않은 내용을 추가하는 경향이 있었다.
- Hall-Petch, Orowan 강화 등 근거가 제시되지 않은 내용을 포함하였다.
- 출력 형식을 일부 준수하지 못하였다.

---

# 11. 최종 선정 모델

## 선정 모델

**ChatGPT (GPT-5.5)**

### 선정 이유

본 업무 과업은 연구보고서 개요를 반복적으로 생성하는 업무이므로 **형식의 일관성**, **환각 억제**, **재사용성**이 가장 중요하다.

ChatGPT는 동일한 프롬프트에서도 가장 안정적으로 출력 형식을 유지하였으며, 정보가 부족한 경우 "확인 필요"를 제시하여 환각 발생 가능성을 낮추었다.

또한 연구 목적, 핵심 키워드, 목차 및 기대효과를 일관된 구조로 작성하여 실제 업무 자동화에 가장 적합한 모델로 판단하였다.

---

# 12. 무료 모델만 사용할 경우의 전략

무료 모델만 사용할 경우에는 출력 품질을 유지하기 위해 다음과 같이 프롬프트를 개선한다.

- 출력 형식을 구체적으로 지정한다.
- 반드시 포함해야 하는 항목을 번호로 제시한다.
- 정보가 부족하면 확인 질문을 먼저 수행하도록 한다.
- 실제 논문이나 실험 결과를 추측하지 말 것을 명시한다.
- 최종 결과만 출력하도록 제한하여 불필요한 내용을 줄인다.

---

# 13. 결론

세 모델 모두 연구보고서 개요를 생성할 수 있었으나 업무 자동화 관점에서는 차이가 있었다.

Claude는 전공 전문성이 우수하였고 Gemini는 다양한 아이디어를 제시하였으나, 형식 일관성과 환각 억제 측면에서는 ChatGPT가 가장 안정적이었다.

따라서 연구보고서 개요 자동 생성 업무에서는 ChatGPT(GPT-5.5)를 최종 모델로 선정하였다.
