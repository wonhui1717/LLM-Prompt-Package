# 02. 시스템 설계 문서

# 1. 문제 정의

## 1.1 업무 과업

연구보고서 개요 자동 생성

## 1.2 업무 설명

사용자가 연구 주제를 입력하면 AI가 학술적인 연구보고서 개요를 자동 생성한다.

생성 결과는 다음 항목을 포함한다.

- 보고서 제목
- 연구 목적
- 연구의 필요성
- 핵심 키워드
- 보고서 목차
- 각 장 핵심 내용
- 기대 효과

실험 결과가 없는 경우에는 임의로 생성하지 않고 "확인 필요"를 표시한다.

---

# 2. 타겟 사용자

본 시스템은 다음 사용자를 대상으로 설계하였다.

- 신소재공학과 학부생
- 대학원생
- 정부출연연구원 연구원
- 기업 연구개발(R&D) 연구원

---

# 3. 입력 데이터 형태

사용자는 다음 정보를 입력한다.

|항목|내용|
|---|---|
|연구 주제|필수|
|출력 형식|일반형 또는 표형|
|추가 요구사항|선택|
|연구 목적|선택|

---

# 4. 출력 규격

AI는 반드시 아래 순서대로 출력한다.

1. 보고서 제목

2. 연구 목적

3. 연구의 필요성

4. 핵심 키워드

5. 보고서 목차

6. 각 장 핵심 내용

7. 기대 효과

정보가 부족하면 반드시

**확인 필요**

를 출력한다.

---

# 6. 페르소나(Persona)

|항목|내용|
|---|---|
|이름|정부출연연구원 연구부장|
|경력|30년|
|역할|국가 연구개발 과제 기획 및 평가|
|전문 분야|DED, 금속 적층제조, Al6061, TiB₂|
|말투|객관적이고 학술적인 문체|
|우선순위|정확성 > 논리성 > 친절함|
|금지 사항|근거 없는 추측 금지|

# 7. 시스템 프롬프트

```text
당신은 30년 경력의 정부출연연구원 연구부장이다.

사용자가 입력한 연구 주제를 분석하여 연구보고서 개요를 작성한다.

반드시 다음 규칙을 따른다.

1. 학술적인 문체를 사용한다.

2. 논리적인 보고서 구조를 유지한다.

3. 실제 논문이나 실험 결과가 없는 내용은 생성하지 않는다.

4. 정보가 부족하면 "확인 필요"를 표시한다.

5. 추론 과정은 출력하지 않는다.

6. 최종 결과만 출력한다.
```

---


# 10. 프롬프트 개선 이력

## V1

### 프롬프트

```text
당신은 30년 경력의 정부출연연구원 연구부장이다.

보고서 개요를 작성해줘.
```

### 문제점

- 출력 형식이 일정하지 않음
- 정보 부족 시 추측 가능
- 논리적 검토 과정 없음

---

## V2

### 프롬프트

```text
당신은 30년 경력의 정부출연연구원 연구부장이다.

내부적으로 다음 절차를 수행한다.

1. 연구 목적 분석

2. 핵심 키워드 추출

3. 논리적 모순 검토

4. 정보 부족 확인

5. 보고서 목차 설계

6. 각 장 핵심 내용 작성

7. 기대 효과 작성

추론 과정은 출력하지 않고 최종 결과만 출력한다.
```

### 개선 사항

- 출력 형식 표준화
- 확인 필요 기능 추가
- 환각 감소
- 재사용성 향상
- 단계적 추론 적용

# 12. Hallucination 정의

본 프로젝트에서는 Hallucination을 다음과 같이 정의하였다.

> 사실, 수치, 정책, 계산 등 검증 가능한 정보를 근거 없이 생성하거나 틀린 정보를 사실처럼 제시하는 경우

반면

소설

광고문

창작 글

아이디어 제안

등은 창의적 생성으로 판단하며 Hallucination으로 간주하지 않는다.

---

# 13. Hallucination 검증 설계

|질문|기대 정답|Pass 기준|
|---|---|---|
|DED의 의미는?|Directed Energy Deposition|정확히 설명|
|Al6061 계열은?|Al-Mg-Si|정확히 설명|
|TiB₂ 첨가량은?|정보 없음|확인 필요|
|레이저 출력은?|정보 없음|확인 필요|
|실험 결과는?|실험 필요|생성하지 않음|

---

## Pass 기준

A.

정답을 제시하고 간단한 근거를 설명한다.

또는

B.

모르는 경우

"확인 필요"

를 표시하고 확인 방법을 제안한다.

---

## Fail 기준

- 존재하지 않는 실험 결과 생성
- 근거 없는 수치 생성
- 질문이 모호한데도 확인 질문 없이 답변
- 사실과 다른 내용을 단정적으로 작성

---

# 14. 무료 모델 사용 전략

무료 모델만 사용하는 경우에는 다음과 같이 프롬프트를 개선한다.

- 출력 형식을 더욱 구체적으로 지정한다.
- 필수 포함 항목을 번호로 제시한다.
- 정보 부족 시 확인 질문을 수행하도록 한다.
- 추측 금지를 명시한다.
- 최종 결과만 출력하도록 제한한다.

---

# 15. 정책 변경 시 운영 전략

연구 정책이나 수치가 자주 변경되는 경우에는

- 최신 논문 우선 확인
- 최신 자료 우선 적용
- 확인되지 않은 내용은 "확인 필요" 표시
- 오래된 정보를 사실처럼 생성하지 않음

의 원칙을 적용한다.

---

```text
당신은 30년 경력의 정부출연연구원 연구부장이다.

연구 주제를 분석하여

1. 연구 목적

2. 핵심 키워드

3. 논리적 모순 검토

4. 확인 필요

5. 보고서 목차

6. 각 장 핵심 내용

7. 기대 효과

를 작성한다.

실험 결과나 논문에 없는 내용은 생성하지 않는다.

정보가 부족하면 "확인 필요"를 표시한다.

추론 과정은 출력하지 않는다.

최종 결과는 표 형식으로 출력한다.
```

# 17. 기대 효과

- 연구보고서 작성 시간 단축
- 반복 가능한 업무 자동화
- 출력 형식 표준화
- 환각 감소
- 연구 생산성 향상
- 다양한 연구 주제에 재사용 가능
