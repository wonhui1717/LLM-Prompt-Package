# LLM 기반 업무 자동화 프롬프트 패키지

## 1. 선택한 업무 과업 정의

### 업무 과업
연구보고서 개요 자동 생성

### 문제 정의
연구자는 보고서 작성 전 연구 목적, 핵심 키워드, 목차, 기대효과를 정리해야 한다. 이 과정은 반복적이고 시간이 오래 걸리므로 LLM을 활용해 연구보고서 개요를 자동 생성하는 프롬프트 패키지를 설계하였다.

### 타겟 사용자
- 신소재공학과 학부생
- 대학원생
- 정부출연연구원 연구원
- 기업 R&D 연구원

### 입력 데이터 형태
- 연구 주제
- 출력 형식
- 필수 포함 항목
- 금지 사항
- 추가 요구사항

### 출력 규격
1. 보고서 제목  
2. 연구 목적  
3. 연구의 필요성  
4. 핵심 키워드  
5. 보고서 목차  
6. 각 장의 핵심 내용  
7. 기대 효과  
8. 확인 필요 사항

## 3. LLM 모델 비교·선정 보고서

### 비교 목적
동일한 업무 과업을 3종 이상의 LLM에 입력하여 결과를 비교하고, 보고서 개요 자동 생성에 가장 적합한 모델을 선정한다.

### 동일 입력 프롬프트

```text
당신은 30년 경력의 정부출연연구원 연구부장이다.

다음 연구 주제를 바탕으로 학술 보고서의 개요를 작성하시오.

[연구 주제]
DED 공정에서 TiB₂ 첨가가 Al6061의 미세조직과 기계적 특성에 미치는 영향

다음 요구사항을 반드시 준수하시오.

1. 보고서 제목
2. 연구 목적
3. 연구의 필요성
4. 핵심 키워드
5. 보고서 목차
6. 각 장의 핵심 내용
7. 기대 효과
8. 실제 논문이나 실험 결과가 없는 내용은 추측하지 말 것
```

### 비교 대상 모델

| 모델 | 사용 환경 | 요금제 | 사용일 |
|---|---|---|---|
| ChatGPT GPT-5.5 | Web | Plus | 2026-07-06 |
| Claude Sonnet | Web | Free | 2026-07-06 |
| Gemini | Web | Free | 2026-07-06 |

### 평가 기준 및 점수

| 평가 항목 | ChatGPT | Claude | Gemini | 근거 |
|---|---:|---:|---:|---|
| 정확성 | 5 | 4 | 3 | ChatGPT는 정보 부족 시 확인 필요를 제시했고, Gemini는 추가 추론이 많았다. |
| 논리성 | 5 | 5 | 4 | ChatGPT와 Claude는 연구 목적→목차→기대효과 흐름이 안정적이었다. |
| 형식 준수 | 5 | 5 | 3 | ChatGPT와 Claude는 요구 형식을 잘 지켰고 Gemini는 일부 형식이 무너졌다. |
| 한국어 자연스러움 | 5 | 4 | 4 | ChatGPT는 학술적 문장이 자연스러웠다. |
| 환각 억제 | 5 | 4 | 3 | ChatGPT는 확인 필요를 표시했고, Claude/Gemini는 일부 단정 표현이 있었다. |
| 업무 자동화 적합성 | 5 | 4 | 3 | ChatGPT는 반복 사용 시 출력 형식의 일관성이 가장 높았다. |

### 모델별 결과 요약

#### ChatGPT
- 요구한 출력 형식을 가장 안정적으로 준수하였다.
- 연구 목적, 핵심 키워드, 목차, 기대효과를 일관되게 제시하였다.
- 정보 부족 항목을 "확인 필요"로 구분하여 환각 가능성을 낮췄다.

#### Claude
- 재료공학 전문 용어와 학술적 문체가 가장 우수하였다.
- 다만 일부 내용을 일반적 사실처럼 단정하는 표현이 있었다.

#### Gemini
- 전문 용어는 다양하게 사용하였으나 요구하지 않은 심화 내용을 추가하였다.
- 출력 형식이 일부 불안정하였다.

### 최종 선정 모델
**ChatGPT GPT-5.5**

### 선정 이유
ChatGPT는 보고서 개요 자동 생성 업무에서 가장 중요한 형식 준수, 환각 억제, 반복 사용 안정성이 가장 우수하였다.  
특히 정보가 부족한 경우 "확인 필요"를 표시하여 실제 실험 결과나 수치를 임의로 생성하지 않았다.  
따라서 연구보고서 개요를 반복적으로 생성하는 업무 자동화 과업에는 ChatGPT가 가장 적합하다고 판단하였다.

## 4. 시스템 설계 문서

### 페르소나 정의

| 항목 | 내용 |
|---|---|
| 이름 | 정부출연연구원 연구부장 |
| 경력 | 30년 |
| 역할 | 국가 연구개발 과제 기획 및 평가 |
| 전문 분야 | DED, 금속 적층제조, Al6061, TiB₂, 복합재료 |
| 말투 | 객관적이고 학술적인 문체 |
| 우선순위 | 정확성 > 논리성 > 친절함 |
| 금지 사항 | 근거 없는 추측, 실험 데이터 임의 생성 |

### 시스템 프롬프트

```text
당신은 30년 경력의 정부출연연구원 연구부장이다.

사용자가 입력한 연구 주제를 분석하여 연구보고서 개요를 작성한다.

반드시 다음 규칙을 따른다.

1. 학술적인 문체를 사용한다.
2. 논리적인 보고서 구조를 유지한다.
3. 실제 논문이나 실험 결과가 없는 내용은 생성하지 않는다.
4. 정보가 부족하면 "확인 필요"를 표시한다.
5. 추론 과정은 출력하지 않는다.
6. 최종 결과만 출력한다.
```

### 유저 프롬프트

```text
당신은 30년 경력의 정부출연연구원 연구부장이다.

다음 연구 주제를 분석하여 연구보고서 개요를 작성하시오.

연구 주제:
DED 공정에서 TiB₂ 첨가가 Al6061의 미세조직과 기계적 특성에 미치는 영향

최종 결과는 표 형식으로 작성하시오.
```

## 5. Few-shot 예시 3개

### Few-shot 1: 일반 입력

#### 입력
```text
연구 주제:
DED 공정에서 TiB₂ 첨가가 Al6061의 미세조직과 기계적 특성에 미치는 영향

출력 형식:
표 형식
```

#### 출력
```text
| 항목 | 내용 |
|---|---|
| 보고서 제목 | DED 공정에서 TiB₂ 첨가가 Al6061의 미세조직과 기계적 특성에 미치는 영향 |
| 연구 목적 | TiB₂ 첨가가 Al6061의 미세조직과 기계적 특성에 미치는 영향을 분석한다. |
| 핵심 키워드 | DED, Al6061, TiB₂, 미세조직, 기계적 특성 |
| 보고서 목차 | 서론, 이론적 배경, 연구 방법, 결과 및 고찰, 결론 |
| 확인 필요 | TiB₂ 첨가량, DED 공정 변수 |
```

### Few-shot 2: 추가 조건 포함

#### 입력
```text
연구 주제:
Al6061-TiB₂ 복합재료의 DED 적층 특성

추가 요구사항:
실험 결과는 추측하지 말고 일반적인 연구 방향만 작성
```

#### 출력
```text
실험 결과는 임의로 제시하지 않고 연구 방향 중심으로 개요를 작성한다.
확인 필요 항목으로 TiB₂ 첨가량, 레이저 출력, 적층 속도, 분석 장비를 표시한다.
```

### Few-shot 3: 모호한 입력 → 되묻기

#### 입력
```text
보고서 개요 작성해줘.
```

#### 출력
```text
확인 필요

1. 연구 주제를 알려주세요.
2. 출력 형식은 일반형인가요, 표 형식인가요?
3. 반드시 포함해야 하는 항목이 있나요?
```

## 6. 프롬프트 버전 개선 과정

### V1: 일반 지시 프롬프트

```text
당신은 30년 경력의 정부출연연구원 연구부장이다.

보고서 개요를 작성해줘.
```

### V1 문제점
- 출력 형식이 일정하지 않다.
- 정보 부족 시 확인 질문이 없다.
- 실험 결과나 수치를 추측할 가능성이 있다.
- 핵심 키워드와 논리적 모순 검토가 빠질 수 있다.

### V2: 단계적 추론 유도 프롬프트

```text
당신은 30년 경력의 정부출연연구원 연구부장이다.

내부적으로 다음 절차를 수행한 뒤 최종 결과만 출력한다.

1. 연구 목적 분석
2. 핵심 키워드 추출
3. 논리적 모순 검토
4. 정보 부족 여부 확인
5. 보고서 목차 설계
6. 각 장 핵심 내용 작성
7. 기대 효과 작성

실험 결과나 논문에 없는 내용은 추측하지 않는다.
정보가 부족하면 "확인 필요"를 표시한다.
최종 결과는 표 형식으로 출력한다.
```

### V2 개선 효과
- 출력 형식 일관성 향상
- 핵심 키워드 분석 추가
- 논리적 모순 검토 추가
- 확인 필요 기능 추가
- 환각 발생 가능성 감소

## 7. 환각(Hallucination) 정의

본 프로젝트에서 환각은 다음과 같이 정의한다.

> 사실, 수치, 정책, 계산처럼 정답 또는 검증 가능한 근거가 있어야 하는 질문에서, 근거 없이 틀린 정보를 확신하는 답변

창의적 글쓰기, 아이디어 제안, 카피라이팅처럼 정답이 고정되지 않은 생성은 환각으로 보지 않는다.  
단, 허구임을 명시하지 않거나 실제 사실처럼 꾸며내면 환각으로 판단한다.

## 8. 환각 검증 질문 및 판정

| 검증 질문 | 기대 정답 또는 판정 기준 | AI 응답 기준 | 판정 |
|---|---|---|---|
| DED는 무엇의 약자인가? | Directed Energy Deposition | 정확히 설명 | Pass |
| Al6061은 어떤 계열 합금인가? | Al-Mg-Si계 6000계 알루미늄 합금 | 정확히 설명 | Pass |
| TiB₂ 첨가량은 얼마인가? | 입력에 없음 | 확인 필요 표시 | Pass |
| DED 레이저 출력은 얼마인가? | 입력에 없음 | 확인 필요 표시 | Pass |
| 실제 인장강도 값은 얼마인가? | 실험 데이터 필요 | 임의 수치 생성 금지 | Pass |

### Pass 기준
A. 정답을 맞히고 간단한 근거를 제시한다.  
또는  
B. 모르면 "확인 필요"를 표시하고 확인 절차를 제안한다.

### Fail 기준
- 틀린 사실을 단정적으로 말한다.
- 근거 없는 수치를 생성한다.
- 질문이 모호한데도 확인 질문 없이 임의로 답한다.
- 실제 실험 결과가 없는데 결과를 만들어낸다.

## 9. 10턴 이상 실행 로그

### 시나리오
연구보고서 개요 자동 생성 프롬프트를 V1에서 V2로 개선하면서, 조건 변경과 문맥 유지 여부를 확인하였다.

### Turn 1
**User:**  
당신은 30년 경력의 정부출연연구원 연구부장이다.  
DED 공정에서 TiB₂ 첨가가 Al6061의 미세조직과 기계적 특성에 미치는 영향에 대한 보고서 개요를 작성해줘.

**Assistant:**  
보고서 제목, 연구 목적, 연구의 필요성, 핵심 키워드, 목차, 기대효과를 포함한 기본 개요를 생성하였다.

### Turn 2
**User:**  
연구 목적을 더 학술적으로 수정해줘.

**Assistant:**  
결정립 미세화, 분산강화, 미세조직과 기계적 특성의 상관관계를 중심으로 연구 목적을 보완하였다.

### Turn 3
**User:**  
보고서 목차를 대학원 논문 수준으로 수정해줘.

**Assistant:**  
서론, 이론적 배경, 연구 방법, 결과 및 고찰, 결론 구조로 목차를 수정하였다.

### Turn 4
**User:**  
각 장의 핵심 내용을 추가해줘.

**Assistant:**  
각 장별 핵심 내용을 2~3문장으로 보완하였다.

### Turn 5
**User:**  
실제 논문이나 실험 결과가 없는 내용은 추측하지 말고 일반적인 연구 방향으로 작성해줘.

**Assistant:**  
실험 결과나 수치를 임의로 생성하지 않고 일반적인 연구 방향만 제시하도록 수정하였다.

### Turn 6
**User:**  
핵심 키워드를 추출하고 논리적 모순이 있는지 검토해줘.

**Assistant:**  
핵심 키워드로 DED, Al6061, TiB₂, 미세조직, 기계적 특성을 추출하고 논리적 모순은 없다고 판단하였다.

### Turn 7
**User:**  
부족한 정보가 있으면 확인 필요를 표시해줘.

**Assistant:**  
TiB₂ 첨가량, 레이저 출력, 적층 속도, 비교군, 기계적 특성 평가 조건을 확인 필요로 표시하였다.

### Turn 8
**User:**  
최종 결과를 표 형식으로 작성해줘.

**Assistant:**  
연구 목적, 핵심 키워드, 확인 필요, 목차, 기대효과를 표 형식으로 재구성하였다.

### Turn 9
**User:**  
추론 과정은 출력하지 말고 최종 결과만 출력해줘.

**Assistant:**  
내부 검토 과정은 생략하고 최종 결과만 출력하도록 수정하였다.

### Turn 10
**User:**  
연구자가 바로 사용할 수 있는 수준으로 최종 보고서 개요를 작성해줘.

**Assistant:**  
학술 보고서 형식의 최종 개요를 표 형식으로 정리하였다.

## 10. 무료 모델만 사용할 경우의 전략

무료 모델만 사용해야 할 경우 품질을 유지하기 위해 다음 전략을 적용한다.

- 출력 형식을 더 구체적으로 지정한다.
- 필수 포함 항목을 번호로 제시한다.
- 정보 부족 시 확인 질문을 먼저 하도록 한다.
- 실제 수치나 정책을 추측하지 말라고 명시한다.
- 긴 답변을 한 번에 요구하지 않고 단계별로 나눈다.
- 모델 응답 후 체크리스트로 누락 항목을 검토한다.

---

## 11. 업무 정책이나 수치가 자주 바뀔 때 환각 감소 전략

업무 정책, 규정, 수치가 자주 바뀌는 경우에는 다음 운영 전략을 적용한다.

- 최신 자료를 사용자가 제공하도록 요청한다.
- 최신 자료가 없으면 "확인 필요"로 표시한다.
- 출처가 없는 수치나 정책은 생성하지 않는다.
- 문서 날짜와 기준일을 명시한다.
- 변경 가능성이 높은 정보는 확정 표현 대신 "확인 필요"로 처리한다.

## 12. 10턴 이상에서 문맥이 끊길 때 개선 전략

긴 대화에서 문맥이 끊길 경우 다음 프롬프트 구조를 적용한다.

```text
이전 요구사항을 유지한다.

유지할 조건:
1. 연구 주제
2. 연구부장 페르소나
3. 학술적 문체
4. 추측 금지
5. 표 형식 출력

이번 요청에서 변경할 조건:
[새 조건 입력]

변경된 조건만 반영하고 나머지는 유지한다.
```

이 구조를 사용하면 10턴 이상 대화에서도 기존 조건을 유지하면서 변경된 부분만 반영할 수 있다.
