# 03. 실행 로그

# 1. 실행 시나리오

## 업무 과업

연구보고서 개요 자동 생성

## 연구 주제

**DED 공정에서 TiB₂ 첨가가 Al6061의 미세조직과 기계적 특성에 미치는 영향**

본 실행 로그는 동일한 연구 주제에 대해 프롬프트를 V1에서 V2로 개선하면서 출력 품질, 문맥 유지, 환각(Hallucination) 감소 효과를 검증하기 위해 수행하였다.

# 3. 실행 로그 (10 Turn)

## Turn 1

### User

당신은 30년 경력의 정부출연연구원 연구부장이다.

다음 연구 주제에 대한 보고서 개요를 작성해줘.

### Assistant

보고서 제목, 연구 목적, 연구 필요성, 핵심 키워드, 보고서 목차를 포함한 기본 개요를 생성하였다.

---

## Turn 2

### User

연구 목적을 조금 더 학술적으로 수정해줘.

### Assistant

결정립 미세화와 기계적 특성의 상관관계를 중심으로 연구 목적을 수정하였다.

---

## Turn 3

### User

목차를 대학원 논문 수준으로 변경해줘.

### Assistant

서론, 이론적 배경, 연구방법, 결과 및 고찰, 결론 구조로 수정하였다.

---

## Turn 4

### User

각 장의 핵심 내용을 추가해줘.

### Assistant

각 장별 핵심 내용을 작성하여 보고서 개요를 보완하였다.

---

## Turn 5 (조건 변경)

### User

실제 논문에 없는 내용은 절대 추측하지 말고 작성해줘.

### Assistant

실험 결과와 수치는 생성하지 않고 일반적인 연구 방향만 기술하도록 수정하였다.

---

## Turn 6

### User

핵심 키워드를 추출하고 논리적 모순을 검토해줘.

### Assistant

핵심 키워드를 추출하였으며 논리적 모순은 없다고 판단하였다.

---

## Turn 7

### User

정보가 부족하면 확인 필요를 표시해줘.

### Assistant

다음 항목을 "확인 필요"로 제시하였다.

- TiB₂ 첨가량
- 레이저 출력
- 스캔 속도
- 비교군
- 기계적 특성 평가 조건

---

## Turn 8

### User

최종 결과를 표 형식으로 작성해줘.

### Assistant

전체 결과를 표 형식으로 재구성하였다.

---

## Turn 9

### User

추론 과정은 출력하지 말고 최종 결과만 출력해줘.

### Assistant

최종 결과만 출력하도록 수정하였다.

---

## Turn 10

### User

연구자가 바로 사용할 수 있는 최종 보고서 개요를 작성해줘.

### Assistant

최종 보고서 개요를 작성하고 학술 보고서 형식으로 정리하였다.

# 6. 환각(Hallucination) 검증

## 환각 정의

본 프로젝트에서는 **사실이나 수치를 근거 없이 생성하는 경우를 환각(Hallucination)**으로 정의하였다.

창의적인 문장 생성은 환각으로 판단하지 않는다.

---

## 검증 질문

|질문|기대 정답|AI 응답|판정|
|---|---|---|---|
|DED는 무엇의 약자인가?|Directed Energy Deposition|정확히 설명|Pass|
|Al6061은 어떤 합금인가?|Al-Mg-Si계 6000계|정확히 설명|Pass|
|TiB₂ 첨가량은?|제공되지 않음|확인 필요|Pass|
|레이저 출력은?|정보 없음|확인 필요|Pass|
|실험 결과는?|실험 필요|생성하지 않음|Pass|

---

## Pass 기준

- 정답과 간단한 근거를 제시한다.
- 모르는 정보는 "확인 필요"를 표시한다.

## Fail 기준

- 존재하지 않는 실험 결과를 생성한다.
- 근거 없는 수치를 생성한다.
- 질문이 모호한데도 임의로 답변한다.

# 8. 무료 모델만 사용하는 경우의 개선 전략

무료 모델만 사용할 경우에는 다음과 같이 프롬프트를 보완한다.

- 출력 형식을 명확하게 지정한다.
- 반드시 포함해야 하는 항목을 번호로 제시한다.
- 정보 부족 시 확인 질문을 수행하도록 한다.
- 추측 금지 규칙을 명시한다.
- 최종 결과만 출력하도록 제한한다.

---

# 9. 정책이나 수치가 자주 변경되는 경우의 운영 전략

- 최신 논문이나 연구자료를 우선 참고한다.
- 최신 정보가 없는 경우 "확인 필요"를 표시한다.
- 사용자가 제공한 최신 자료를 우선 적용한다.
- 오래된 정보를 사실처럼 생성하지 않는다.

---

# 10. 문맥 유지 개선 전략

10턴 이상의 대화에서는 이전 요구사항을 유지하면서 변경된 조건만 수정하도록 프롬프트를 설계하였다.

이를 위해 다음 규칙을 적용하였다.

- 이전 요구사항 유지
- 변경된 조건만 수정
- 출력 형식 유지
- 페르소나 유지
- 최종 결과만 출력

---

# 11. 최종 결과 요약

본 프로젝트에서는 연구보고서 개요 자동 생성 업무를 대상으로 프롬프트를 V1에서 V2로 개선하였다.

V2 프롬프트는 연구 목적 분석, 핵심 키워드 추출, 논리적 모순 검토, 확인 필요 기능, 표 형식 출력 규칙을 추가하여 결과의 논리성과 재사용성을 향상시켰다.

또한 실제 논문이나 실험 결과가 없는 정보는 생성하지 않고 "확인 필요"를 표시하도록 설계하여 환각 발생 가능성을 줄였다.

최종적으로 연구자가 반복적으로 사용할 수 있는 업무 자동화 프롬프트를 구축하였다.
